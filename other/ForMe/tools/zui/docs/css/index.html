CSS开发规范(CSS Code Guideline)

一、CSS浏览器支持标准

  1. 标准－交互和视觉完全符全设计的要求
  2. 普通－视觉上允许有所差异，不破坏页面整体效果
  3. 简洁－可忽视视觉上的问题，但不防碍使用

二、尽可能的通过继承和层叠重用已有样式

三、根据新建样式的适用范围分为四级：全站级、产品级、页面级、插件级

  1. 目前全站级的CSS文件有reset-layout.css（布局）和siteStyle.css
  2. 产品级CSS是指作用于某一垂直产品，文件放在css/biz下
  3. 页面级指仅在一个或少量几个页面中用到。如果仅在一个页面中用到的采用内联方式嵌入于页面head里，多于两个页面的放到外联的CSS文件中，该文件放在css/biz下
  4. 插件级是指页面中用到的特殊交互效果的css，该文件放在css/widget下
  5. 内联和外联的CSS都必须放在页面的head里，顺序是：全站级、产品级、页面级、插件级
  6. 改动全站级CSS，需经过全面测试

四、CSS规则的书写格式要求

  1. 属性需要写在一行，如：
    2.         .selector { property:value;property:value; }
  2. 多个selector并列写，“逗号”后加一个空格：
    3.         .selector1, .selector2, .selector3 { property:value;property:value; }
  3. 兼容多个浏览器时，将标准属性写在后面，如：
    4.         -webkit-border-radius:4px;-moz-border-radius:4px;border-radius: 4px;

五、ID和Class命名。命名不要用缩写，单词间用"-"做为连接符

  1. ID是用来标识具体模块，命名必须具体且唯一，由前缀和名字组成。不要滥用ID
  2. Class是用来标识某一类型的元素，命名简洁表意清楚
  3. 推荐使用的class名：

    表示状态: .selected, .open, .collapsible, .active, .on, .current
    表示位置: .first, .last, .main, .side
    表示结构: .hd, .bd, .ft, .col, .section
    通用元素: .tb, .frm, .nav, .list, .item, .tag, .pic, .info

六、尽量避免使用CSS Hack

  1. 区别属性：

    IE6: _property:value
    IE6/7: *property:value
    IE6/7/8/9: property:value\9
    非IE6: property//:value

  2. 区别规则：

    IE6: * html selector { ... }
    IE7: *:first-child+html selector { ... }
    非IE6: html>body selector { ... }
    firefox only: @-moz-document url-prefix() { ... }
    saf3+/chrome1+: @media all and (-webkit-min-device-pixel-ratio:0) { ... }
    opera only: @media all and (-webkit-min-device-pixel-ratio:10000),not all and (-webkit-min-device-pixel-ratio:0) { ... }
    iPhone/mobile webkit: @media screen and (max-device-width: 480px) { ... }

七、CSS和图片不能使用的命名

  1. css样式和图片表示长乘宽不能用75x56，统一命名为75-56；百分比不能1/5，统一成1size5
  2. 图片的命名中不能包含ads的名称

css注释

  1. 头部拟写规范:
    2.         
            /**
             * 名称：
             * 内容描述：
             * 
             * [TOC]
             */
  2. 注释：
    3.           /***********************************************************************************************************
           * 整段注释
           ***********************************************************************************************************/

           /* 行间注释 */
